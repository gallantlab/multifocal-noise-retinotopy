User's guide

Operating manual and technical reference: the interface, every parameter with its exact units, the design and synthesis methods, and the limits that matter when designing a run. New to retinotopy? Start with the Beginner's guide.

Overview

The stimulus is a disc of diameter width split into N regions — angular wedges (polar-angle mapping) or concentric rings (eccentricity mapping). Independent maximum-length sequences (m-sequences) drive a multifocal design: one gates each region on/off across 63 states, a second sets each on-region's noise orientation, and an optional third sets the whole-field background orientation. Each on-region carries independent multicolour bandpass 1/f spatiotemporal noise of a single orientation. States fade to the background; the movie is optionally padded with full-screen isotropic noise. Output is a PNG frame sequence plus a complete JSON design record and verification plots.

terminology

"Region" = a wedge or a ring. The parameter keys keep their historical n_wedges / wedge_* names for both geometries; the UI relabels them ("# rings", "ring duration", …) when you select rings.

Running it

python3 server.py            # stdlib HTTP server on :8000
# open http://localhost:8000

Requirements: Python 3 with numpy, Pillow, matplotlib. The page must be served (generation calls the backend) — opening index.html as a file:// shows a notice and will not generate. A long run can be aborted with Cancel (stops within roughly one state's compute).

The interface

Left panel: the parameter form, Generate, and (while running) Cancel with a progress bar. The rest is the viewer, rebuilt after each run from movie_meta.json:

  • Movie — canvas player (Pause/Play) looping the rendered frames at the true frame rate. Readout shows frame, time, the current state (or padding), and the on-regions with their orientations. Up to 900 frames are preloaded for playback.
  • On/off m-sequence — the design matrix (states down × regions across, labelled W1… or R1…; red = on) with a time axis and a cursor tracking playback.
  • Orientation m-sequence — orientation index per state × region, one hue per orientation (off-regions dimmed).
  • Noise spectra — measured vs. ideal temporal, spatial (radial + 2-D), and orientation spectra, sampled from the first non-empty state, for verification.

Design & m-sequences

Up to three binary maximum-length sequences of order 6 (period L = 2⁶−1 = 63), generated by a Fibonacci LFSR from distinct primitive polynomials:

  • On/off (region location): x⁶+x+1 (taps [6,1]).
  • Orientation (per-region foreground orientation): x⁶+x⁴+x³+x+1 (taps [6,4,3,1]).
  • Background (whole-field background orientation, used only when background = oriented_mseq): x⁶+x⁵+x²+x+1 (taps [6,5,2,1]).

Region k reads the on/off and orientation sequences at the same circular shift round(k·L/N), making the region regressors near-orthogonal (the basis of the multifocal method). Because both are read at that same shift, their per-region correlation is just the zero-shift correlation of the two base sequences (≈ −0.02), so location and orientation are separately estimable in an encoding model. The background, however, is a single stream at one fixed shift (not per region): its correlation with a given region's regressor varies with that region's shift — ≈ −0.02 for most regions but up to ≈ 0.24 at the defaults (worst case ≈ 0.27/0.37 over all shifts). Still modest and separable, but a distinct primitive polynomial alone does not guarantee the tight decorrelation of the same-shift foreground pair. The orientation index is decoded from ceil(log₂K) consecutive bits, taken mod K — exact for K=2 (see Orientation). The background sequence drives a single whole-field stream decoded the same way (see Background orientation count).

Noise synthesis

Each noise block is white Gaussian noise shaped in the 3-D Fourier domain by a separable amplitude filter H = spatial(x,y) × temporal(t):

noise = irfftn( rfftn(white) * H )      # over (x, y, t)
noise /= noise.std()                    # unit variance
pixel = clip(128 + 38 * noise, 0, 255)  # BG=128, GAIN=38 gray-levels/std
  • Spatial filter: radial bandpass [sf_lo, sf_hi] cyc/width with 1/f or flat amplitude, optionally masked to one orientation.
  • Temporal filter: bandpass [tf_lo, tf_hi] Hz, DC zeroed.
  • Colour: 3 independent fields through the same H (each Fourier component a random RGB); BW replicates one field.
  • Independence: every region uses its own seed (BASE_SEED + s·N + k), so adjacent regions do not blend — a deliberate seam at shared boundaries keeps them separable.

Geometry

Both geometries partition the disc r ≤ width/2 with pixel centres at i+0.5; the masks tile the disc with no gaps or overlaps.

Wedges wedges

Angles measured from 3 o'clock CCW; wedge k spans [k, k+1)·360/N°, with all boundaries offset by wedge rotation.

Rings rings

Ring k is the annulus [b_k, b_{k+1}) (outer ring takes the disc edge), with boundary radii b_k from 0 (fovea) to R = width/2:

SpacingBoundary radiiCharacter

Caveats & limits

Temporal band

The highest representable temporal frequency is the Nyquist fps/2 (15 Hz at 30 fps); the UI does not clamp this, so keep tf_hi ≤ fps/2. The lowest is set by the state window, ≈ 1/wedge_sec (≈ 0.24 Hz at 4.1 s).

Spatial band & auto-scaling

cyc/width = cycles across the movie width = the integer FFT index for a width-px field; spatial Nyquist is width/2. Because the unit is tied to the pixel width, the UI auto-rescales the SF band by the width ratio when you change width (constant cycles-per-pixel), which keeps the texture scale fixed and the band in range — e.g. 512→256 turns 2/128 into 1/64. Edit SF after setting width to override.

Orientation

All orientations are selected by the same thin angular wedge of half-width min(4°, 90/2K°) — i.e. 4° until K ≥ 12. Cardinals (0°/90°) fall on the FFT grid and could be kept as an exact infinitely-narrow Fourier line, but that makes them 1-D and perfectly coherent along the bar while obliques (off the grid) are 2-D textures — a coherence confound on top of orientation. One finite wedge for every angle matches their spatial bandwidth, so only orientation varies. The K-ary orientation decode (mod K of ceil(log₂K) bits) is a true balanced m-sequence only for K=2; powers of two stay close to balanced, other K are more biased. Measured per-orientation counts over 63 states: K=2 → 248/256; K=4 → 120/128/128/128.

Background orientation count

With the oriented background, the angle each state is the midpoint of the largest empty arc among the on-regions' orientations (maximally orthogonal). Outputs lie on a 90/K° grid, so at most 2K values are possible, but fewer occur. Distinct background angles over 63 states:

More regions → fewer distinct background angles: with many regions on it's likely all K orientations are present, and the full set always maps to the single value 90/K°. Use fewer regions or larger K for a background that varies. Defaults (8 regions, K=2) → 3 angles. All 63 per-state angles are recorded in bg_orient.

The oriented_mseq background sidesteps this region-count dependence entirely: a third m-sequence from a distinct primitive polynomial (x⁶+x⁵+x²+x+1) drives a single whole-field orientation per state, decoded mod K exactly like the foreground. Its K angles are the foreground set rotated half a step (90/K°) so the two interleave instead of coinciding (K=2: foreground 0°/90° → background 45°/135°; K=1: 0° → 90°). You therefore cycle through all K background orientations regardless of how many regions are on — counts over 63 states are near-balanced for powers of two (K=2 → 31/32; K=4 → 15/16/16/16) and more uneven otherwise (K=3 → 31/16/16), the same mod K bias as the foreground. The angle set is recorded in bg_angles, the per-state angles in bg_orient.

Other notes

  • Region cap: N is clamped to [1, 63] — beyond 63 regions cannot get distinct shifts (collinear regressors).
  • Fades are to background, not crossfades: states do not overlap; the screen returns to the background between them.
  • Reproducibility: all RNG seeds derive from a fixed BASE_SEED, so identical parameters reproduce an identical movie.
  • Padding & random background are isotropic (no orientation filtering).
